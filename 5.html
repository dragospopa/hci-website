Project Overview

Dr John has been given access to vast data which he submitted to CKAN for analysis. Dr John wants to discuss and share his findings with fellow academics and members of the public, and he wants this process to be as simple and professional as possible.

One day, Dr John is told of the services provided by The UDL platform. So Dr John opens a web browser and types UDL platform, at which stage Dr John would make an account or login in through his institution. He then clicks the Login button.

Functional Requirements

The UDL platform will contain an instruction set on how to familiarize himself with the terminal. Dr John will be able to query the dataset using a python terminal. The dataset is already available, since both CKAN and UDL are open source platform, hence he will be able to access them. Dr John then types a python script. UDL platform returns a file containing all relevant information. As he decides he wants to access the file, he clicks open and scrolls down to a graphs file then clicks it. A map or graph appears. Dr John can continue and inspect each sub-file.

Non-functional Requirements

Dr John wants to start a forum. So he clicks Start forum, he chooses his tags and subject title. Then types in his content. Then clicks Go. The forum is now public so other users can choose to reply. Dr John wants to search other datasets that have a particular subject tag so he clicks home button and fills in the search form with title Search. At this point, Dr John is displayed with all forums with those tags.
